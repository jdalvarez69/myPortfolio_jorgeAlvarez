Process

How I work

Every project I've done follows some version of this sequence. Define what I'm building, plan how to build it, build it, check if it's actually good, then deliver it. The steps aren't always clean, but having a consistent structure prevents things from drifting.

  1. 01

    Research

    Understand the problem before doing anything else. For a project, this means reading, asking questions, looking at existing examples, and clarifying what the end state actually needs to look like. For the portfolio, this meant researching what project managers actually do — not surface-level descriptions, but concrete job responsibilities.

  2. 02

    Plan

    Define the scope, break it into concrete tasks, and sequence them in a logical order. Planning forces decisions early. On this site: page structure, content per section, design system, and file organization were all decided before writing a line of code.

  3. 03

    Build

    Execute in steps, completing one piece at a time. I use AI as a collaborative tool during this phase — for generating structure, drafting copy, writing and reviewing code, and thinking through edge cases. AI assists execution; it does not make decisions about direction or quality.

  4. 04

    Review

    Check the work against the original plan. Does it accomplish what it was supposed to? Is the content accurate and honest? Does anything need to be cut or clarified? Review is where I catch inflated language, unclear structure, or technical issues that weren't obvious during the build phase.

  5. 05

    Ship

    Deliver and document. For web projects, this means deployment, validation, and recording what was built and what decisions were made. For future projects, this includes stakeholder handoff and a record that supports maintenance or iteration.

AI in my workflow

How I actually use AI

I use AI tools — primarily GitHub Copilot and ChatGPT — throughout every phase of my projects. For brainstorming and structure, drafting and revising written content, writing and reviewing code, and critiquing what I've built.

AI is not a replacement for judgment. Every decision about scope, direction, tone, content, and quality is mine. AI speeds up execution and helps me work through problems, but I'm responsible for the final output and I review everything before it ships.

This is the same model I intend to carry into project management roles — use available tools effectively, maintain decision ownership, and never hand off critical judgment to a process you can't explain or verify.

Brainstorming

Generating options, challenging assumptions, and exploring directions quickly.

Structure

Organizing content hierarchies, outlining documents, and sequencing tasks.

Writing revision

Cutting inflated language, improving clarity, and checking consistency of tone.

Code collaboration

Writing, reviewing, and debugging HTML, CSS, and JavaScript alongside a human decision loop.

Critique

Asking AI to find weaknesses in what I've built before calling it done.

Quality control

Final decisions and responsibility remain mine. Every output gets reviewed before it ships.
